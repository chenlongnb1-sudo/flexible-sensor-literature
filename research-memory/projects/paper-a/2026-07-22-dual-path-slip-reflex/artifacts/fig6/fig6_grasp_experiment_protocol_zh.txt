一、任务定义

在稳定法向夹持背景下，使用带刻度、底部挂钩的标准圆柱，由砝码/滑轮或电动位移台施加可重复突加载荷。系统使用冻结的 Vz/Vx/Vy 事件阈值完成接触停止、事件唤醒、限时增握或方向纠正，并在相同任务下比较四条采集路径的任务性能与电子能耗。

本实验要证明的内容：
1. ADC 前三矢量和事件接口能够支撑限定的抓取反射任务。
2. A4 检验量化前事件是否相对 A3 减少空闲期嵌入式 ADC 转换与 ADC/reference 能量，并在匹配任务下测量其对 MCU 活跃、通信量和每任务总电子能量的净影响。
3. 完整 Vz/Vx/Vy 相比 Vz-only 能够提供方向相关的扰动信息。

推荐论文主张句（仅在实测满足全部条件时使用）：
Pre-ADC event formation gates digitization during idle intervals and reduces measured electronics energy relative to a matched ADC-first event baseline while preserving the preregistered grasping-performance boundary.

本实验不证明：
1. Vz/Vx/Vy 等价于完整三维力测量。
2. 四通道数字后端不能计算同样的三个矢量。
3. 四通道变三通道本身必然产生大幅节能。
4. 简单抓住多个物体构成系统创新。
5. 只有本系统能够实现事件驱动控制；持续 ADC 配合 DMA、analog watchdog 或低功耗监控同样可以在数字侧产生事件。

二、硬件与测量边界

1. 同一个四单元传感器用于全部路径。物理排列为 45 度菱形：R1 上、R2 左、R3 下、R4 右；逻辑映射固定为 V1=R4、V2=R1、V3=R2、V4=R3。
2. 夹爪：记录型号、驱动方式、闭合速度、位置/速度命令接口和是否具有 x-y 平移能力。
3. 控制分工：P12 锁定前必须区分触觉/事件 MCU 与电机控制器。若同一 MCU 持续输出低层 PWM，Approach 阶段主核不能休眠；若恒速闭合由厂商伺服或独立 motor controller 执行，CONTACT 路径为 comparator -> tactile MCU wake -> stop command -> motor controller。
4. 主对象：使用已生成设计中的带刻度、底部挂钩标准圆柱。挂钩通过固定方向滑轮连接标定砝码或力执行器，使突加载荷的方向、幅值、上升时间和持续时间可重复。禁止手推。
5. 一维夹爪主任务：固定方向突加载荷 -> Vx/Vy 不平衡事件 -> 触觉 MCU 唤醒 -> 限时增握命令 -> 电机控制器执行 -> 比较滑移距离。该配置不得声称 signed x-y correction。
6. 只有平台实际具有 x-y 运动能力时，才扩展为 +x、-x、+y、-y 扰动及反方向小步纠正；四方向结果属于台架/泛化验证。
7. 独立参考：至少具有可追溯的扰动力参考；物体位移优先由圆柱刻度配合同步相机获得，也可使用编码器或位移传感器。
8. 同步采集：参考力、Vz/Vx/Vy、事件 GPIO、ADC_EN/CS/DRDY、MCU_ACTIVE、MOTOR_EN_SAFE、stop/grip command、motor-controller state、物体位移和各电源轨必须共享时钟或共享 TTL 同步脉冲。
9. 诊断 DAQ/示波器/逻辑分析仪可以连续记录全通道，但必须位于“外部观察仪器”边界之外，不计入部署系统能耗。诊断 DAQ 连续采样不等于嵌入式 ADC 持续工作。
10. 正式毫秒级时序使用同步 DAQ、示波器或逻辑分析仪。慢速 USB 轮询仅用于调试，不能用于延迟结论。
11. 触觉电子能量边界必须包含传感器/模拟矢量前端、常开比较器、嵌入式 ADC/reference、触觉 MCU和通信。电机控制器电子轨和执行器功率分别测量、分别报告，不能混入触觉电子系统节能。

三、四条公平比较路径

A1 Raw-4 continuous
- V1-V4 连续进入 ADC。
- MCU 使用相同公式计算 Vz、Vx、Vy。
- ADC、MCU和通信在完整任务时窗内连续工作。

A2 Vector-3 continuous
- 模拟前端在 ADC 前产生 Vz、Vx、Vy。
- ADC 连续采三通道，MCU和通信连续工作。
- A2-vs-A1 只用于解释转换数、数据量和连续路径资源差异。

A3 ADC-first event（强数字事件基线）
- 与 A4 共享同一套 vector AFE、同一 Vz/Vx/Vy 三路 ADC 输入、同一 ADC/reference、活动采样率、阈值物理定义和下游控制器。A3 不得改成 raw-4。
- 嵌入式 ADC 持续采样，或使用实际 ADC/MCU 支持的 DMA、analog watchdog、低功耗监控等最强可用数字方案。
- 事件在量化后产生。触觉 MCU 主核是否可在数字侧事件前休眠，取决于它是否还负责低层电机 PWM；不得在 P12 锁定前无条件假定休眠。
- ADC/MCU 型号和能力仍为 P10/P12 TBC；不得预先假定具有 watchdog、DMA、differential 或 PGA。

A4 Pre-ADC vector event（本方案）
- 与 A3 共享相同 vector AFE、Vz/Vx/Vy ADC 输入、ADC/reference、活动采样率、阈值和控制器；唯一实验变量是事件在量化前还是量化后形成。
- 模拟矢量和常开比较器在量化前产生 CONTACT、X、Y 和 OVERLOAD 事件。
- Hold/无扰动窗口内嵌入式 ADC 停止转换或进入 standby，事件后启动 reference/ADC 并采固定长度 Vz/Vx/Vy burst。Approach 阶段触觉 MCU 是否可休眠取决于电机控制分工。
- A4-vs-A3 用于隔离事件发生在量化前还是量化后的系统差异。
- `ADC_EN/CS/DRDY` 只作为外部活动证据，不能单独证明芯片内部停止转换或 reference 断电。P10 锁定后必须按器件 datasheet 定义 standby/power-down，并用 ADC/reference 分轨电流验证。在此之前统一标注 `standby to be verified`。
- 若实际嵌入式 ADC 仍一直采样，只能写 hardware-thresholded event-driven control，不能写 ADC-gated 或 event-triggered acquisition。

四条路径固定不变的条件：传感器、物体、初始姿态、目标夹持状态、ADC 位数、活动采样率、MCU、活动时钟、数字滤波、阈值物理定义、任务时窗、扰动波形和成功判据。特别是 A3/A4 必须共享同一 vector AFE、三路 ADC 输入、ADC/reference 和下游控制器。

比较解释必须分组：
- A1-vs-A2：隔离 raw-4 到 vector-3 的转换数和数据量变化。
- A3-vs-A4：隔离数字侧事件与量化前事件的位置差异。
- A1/A2-vs-A3/A4：只作为连续工作与事件控制的系统背景，不替代上述成对因果比较。

四、试验前必须冻结的参数

1. theta_c：CONTACT 的 Vz 阈值和持续时间规则。
2. theta_o：由实测 Vz-载荷标定得到的 OVERLOAD 阈值。
3. theta_x、theta_y：Vx/Vy 正负方向事件阈值和去抖规则。
4. 活动采样率、ADC 位数、满量程和参考源。
5. 事件后 burst 长度；建议先用工程预实验确定，再在正式测试前冻结。
6. 夹爪闭合速度、接触后保持命令和最大允许夹持力。
7. 方向纠正步长/速度/持续时间；若只有一维夹爪，冻结事件后增握量。
8. 任务起止时刻、恢复容差和容差持续时间。
9. A3 使用的最强数字低功耗模式、DMA/watchdog 配置和事件生成位置。
10. 低/中/高事件率或空闲占空比条件，用于 break-even 分析。
11. CONTACT 周期中 stop command 与可选 burst 的顺序。
12. 突加载荷周期中“先执行有界动作”或“先短 burst 确认”的唯一正式策略。
13. Vz-only 的最强可用 Vz 绝对阈值和/或变化阈值，及与 Full vectors 完全相同的事件后增握动作。

这些参数直接继承 Fig. 5 的冻结配置。不得按物体、方向或路径重新调阈值。

五、单次试验的标准流程

T0 基线（1-2 s）
- 物体位于固定治具，夹爪未接触。
- 记录各轨静态功率、噪声和误触发。

T1 接近
- 夹爪以固定速度闭合。
- 执行闭合命令的控制器保持所需状态。若触觉 MCU 同时输出低层 PWM，则其主核保持活动；若独立 motor controller/servo 执行恒速闭合，触觉 MCU 可按实测状态休眠或等待。
- A1/A2 连续采集；A3 保持嵌入式 ADC/数字监测链按冻结配置工作；A4 仅保持模拟矢量前端、比较器等规定常开电路，嵌入式 ADC/reference 停止或 standby。

T2 CONTACT 停止
- CONTACT 没有独立硬件 motor gate。CONTACT edge 后由触觉 MCU/controller 唤醒并原子快照事件位，立即向 motor controller 发出 stop/hold command，避免等待完整 ADC/reference settling 和 burst 而增加接触过冲。
- stop/hold command 之后可启动 ADC/reference，等待实测 settling 并采短 burst 用于确认或记录；该 burst 不是 CONTACT 停止的前置条件。
- 记录参考接触时刻、CONTACT GPIO、触觉 MCU 唤醒、stop command、电机控制器响应和实际运动停止时刻。

T3 稳态保持（建议 1 s）
- 检查 Vz 位于允许范围，物体无明显滑移。
- Hold/无扰动期是最明确的低占空比窗口；A3/A4 是否休眠以及休眠深度均由 MCU_ACTIVE、ADC_EN/CS/DRDY 和电源轨实测，不由状态图推断。
- 未达到稳态的试次保留并标记失败，不能删除后重做替代。

T4 随机突加载荷
- 一维夹爪主任务从无加载、低、中、高突加载荷中按预生成随机表选择；砝码/拉力通过圆柱底部挂钩和固定方向滑轮施加。
- 具有 x-y 平台时，可扩展 +x、-x、+y、-y 和无扰动五种条件。
- 扰动力峰值、上升时间、持续时间和加载位置保持一致。

T5 突加载荷 X/Y 事件周期
- 在允许休眠的控制分工和 Hold/无扰动阶段，触觉 MCU 可进入冻结的低功耗状态；X/Y edge 后唤醒并原子快照六个事件位。
- 合法策略 1：先执行冻结的有界增握/纠正命令，再启动 ADC/reference、等待实测 settling、采短 burst 并记录。
- 合法策略 2：先启动 ADC/reference、等待实测 settling、采冻结长度的短 burst 确认，再执行有界命令并记录。
- 预实验后必须只冻结其中一种策略用于正式测试，并实测从事件 edge 到 control command 的延迟。不得按试次或负载切换策略。
- 随后 ADC/reference 回 `standby to be verified`，触觉 MCU 按实际控制职责返回低功耗状态或继续运行。
- 规划中的 OVERLOAD 硬件旁路拟直接控制 MOTOR_EN_SAFE，但当前仅为 `[TO BE VALIDATED]`。在 P13/P14 未锁定且无关断延迟实测前，不能写成已实现的硬件先切断；MCU 后续记录和安全恢复策略同样待驱动接口确认。

T6 恢复动作
- 具有 x-y 运动能力：沿检测方向的反方向执行固定小步纠正。
- 只有一维夹爪：执行冻结的限时增握动作，只能写“突加载荷触发增握”，不能写“带符号方向纠正”。

T7 恢复判定与释放
- 物体位移和参考力进入预设容差并保持规定时间后记为恢复。
- 固定保持时长结束后释放，完整保存所有通道。

六、推荐试验矩阵

主比较（正文最低数据集）
- 1 个带刻度、底部挂钩的标准圆柱。
- 三天 pilot：4 条路径 x 4 个突加载荷条件（无/低/中/高）x 每格 10 次 = 160 次独立抓取，用于可行性、描述性效应和方差估计，不足以直接支持 5 个百分点非劣效结论。
- 正式非劣效样本量必须根据 pilot 基线成功率、冻结 margin、alpha 和 power 重新计算，可能远高于 160 次。
- 按随机区组交错 A1/A2/A3/A4，避免温漂和电池状态与路径顺序混杂。

因果消融（正文 panel e）
- 在 A4 下比较 Full Vz/Vx/Vy 与 Vz-only。
- 2 个控制器 x 4 个突加载荷条件 x 每格至少 10 次 = 80 次。
- Vz-only 不是无事件稻草人：仅用校准集冻结最强可用的 Vz 绝对阈值和/或 Vz 变化阈值作为突加载荷触发规则，事件后的增握动作与 Full vectors 完全相同；正式测试禁止重调。
- 单一固定方向挂钩实验只证明切向不平衡检测与反射，不能证明 signed direction control。

次级泛化（时间允许时）
- 使用几何和尺寸尽量一致的刚性、柔顺、低摩擦圆柱/套筒。
- A4 x 3 个物体 x 4 个条件 x 每格至少 5 次 = 60 次。
- 多物体结果只说明限定条件下的泛化，不替代主比较。

事件率 / break-even（工程边界）
- 使用冻结的任务窗口构造低、中、高事件率或空闲占空比条件。
- 低事件率应包含长空闲和少量 burst；高事件率应覆盖 burst 接近连续工作的边界。
- 使用实测各状态功率和实际事件时间积分 A3/A4 能量，求 A4 与 A3/A2 能量相交的 break-even 点。
- 事件频繁时 A4 可能失去节能优势；必须作为边界报告，不得隐藏。

七、指标定义

1. 接触停止成功：CONTACT 后未超过冻结的峰值夹持力上限，且物体进入稳定保持。
2. 方向事件正确：参考扰动方向与 X/Y 事件符号一致；斜向或边界样本按预注册规则处理。
3. 任务成功：未掉落、未触发安全停止、最大滑移不超过 delta_max，并在 t_recovery_max 内恢复。
4. 峰值夹持力：独立参考力在任务时窗内的最大值。
5. 最大滑移：扰动后物体相对初始稳定位置的最大位移。
6. 事件延迟：参考扰动力越过定义阈值到事件 GPIO 有效沿。
7. 控制延迟：事件 GPIO 有效沿到控制命令或 MOTOR_EN_SAFE 改变。
8. 恢复时间：事件 GPIO 有效沿到位移/参考力进入恢复容差并持续规定时间。
9. MCU 活跃占空比：MCU_ACTIVE 高电平总时长除以固定任务时窗。必须实测，不得根据软件状态图推断；触觉 MCU 与 motor controller 分开报告。
10. ADC 转换数：由 ADC_EN/CS/DRDY 计数，不用软件估算替代硬件计数。
11. 每任务电子能量：E_task = integral V(t)I(t)dt；多电源轨分别积分后求和。
12. 执行器能量：单独积分和报告，不参与触觉电子能量节省百分比。

八、预期的“好结果”（投稿目标，不是已测结果）

1. 候选工程目标：A1、A2、A3、A4 的任务成功率差异落在预先声明的非劣效界限内。5 个百分点只是候选 margin，必须在 pilot 后、正式实验前冻结并重新计算样本量；当前 160 次只做描述性比较。
2. A4 的最大滑移、峰值夹持力和恢复时间不劣于 A1/A2/A3 的预设容差。
3. Full Vz/Vx/Vy 在方向识别、最大滑移或恢复时间上优于 Vz-only，且失败案例支持这一因果解释。
4. A2 相比 A1 主要表现为转换数和数据量变化；若总电子能耗无明显降低，应如实报告。
5. A3 证明数字侧事件基线可以降低 MCU 主核活跃和通信量；不能把事件驱动归为本系统独有能力。
6. A4 相比 A3 的空闲期嵌入式 ADC 转换数进一步下降；只有硬件记录证明 ADC 停止/standby 时，才使用 ADC-gated acquisition 表述。
7. A4 的 E_task 净变化必须实测。30% 只是候选工程目标，需在正式实验前冻结，不是预设结果；若达不到，只能报告实际权衡，不能保留“大幅节能”表述。
8. 无扰动条件不应频繁唤醒；误触发率和漏检率必须与任务性能一起报告。
9. break-even 分析应显示 A4 的优势随事件率上升而缩小，并明确优势消失的工作边界。

九、怎么才算完成

Gate 1 - 硬件可用
- 四条路径可切换并通过同源输入检查；A3 实现实际硬件支持的最强数字低功耗事件模式。
- Fig. 5 阈值配置已冻结并记录版本。
- 事件、ADC/MCU、控制、参考力、运动和电源轨已同步。

Gate 2 - 任务闭环
- CONTACT、突加载荷、无扰动和 OVERLOAD 安全路径均完成台架验证；四向扰动仅在 x-y 平台真实可用时要求完成。
- 只有完成 motor enable/brake 极性、上电/掉电安全状态、故障注入和 overload-to-disable latency 实测后，OVERLOAD 才能从“planned hardware-disable path”升级为结果性硬件保护主张。
- 若声称带符号纠正，机器人必须具有对应 x-y 运动并实际执行反方向动作。

Gate 3 - 数据完整
- 主比较试次完成，失败和安全停止全部保留。
- 每条记录具有唯一的器件、物体、路径、方向和试次 ID。
- 无按结果重新调阈值、删失败或改变任务时窗。

Gate 4 - 量化证据
- 同试次同步时序可重建两个完整因果链：CONTACT -> wake/snapshot -> stop command -> optional burst，以及突加载荷 -> X/Y event -> frozen action/burst policy -> grip command -> 运动恢复。
- 性能统计包含独立试次原始点和区间估计。
- 能耗包含各电子电源轨、测量不确定度和执行器分离结果；ADC/reference standby 由 datasheet 状态定义、外部活动信号和分轨电流共同验证。
